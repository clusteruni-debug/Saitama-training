---

## 🎯 프로젝트 개요

사이타마(원펀맨) 컨셉의 홈트레이닝 앱.
맨몸운동(푸시업, 스쿼트, 턱걸이, 코어) 4개 트랙을 중심으로,
사용자의 수행 데이터를 기반으로 자동으로 갯수를 늘리고
더 어려운 동작 변형으로 업그레이드해주는 **프로그레시브 오버로드 시스템**.

---

## ⚡ 핵심 기능

### 1. 운동 프로그레션 트리 (4갈래: Push / Squat / Pull / Core)
각 트랙마다 5~7단계의 동작 변형. 마스터하면 다음 레벨로 자동 진급.

| 트랙 | 레벨 0 | 레벨 1 | 레벨 2 | 레벨 3 | 레벨 4 | 레벨 5 |
|------|--------|--------|--------|--------|--------|--------|
| **Push** | 무릎 푸시업 | 일반 푸시업 | 와이드 푸시업 | 다이아몬드 | 아처 | 원암 |
| **Squat** | 하프 스쿼트 | 풀 스쿼트 | 불가리안 | 점프 스쿼트 | 피스톨(보조) | 피스톨 |
| **Pull** | 인버티드 로우 | 네거티브 턱걸이 | 턱걸이 | 친업 | 와이드 풀업 | 머슬업 |
| **Core** | 플랭크(초) | 니 레이즈 | 레그 레이즈 | 행잉 니레이즈 | 행잉 레그레이즈 | 드래곤 플래그 |

### 2. 자동 볼륨 조절 (RPE 피드백 기반)
- 운동 후 **"쉬웠다 / 적당했다 / 힘들었다"** 3단계 피드백
- 다음 세션 자동 조절: 쉬움 → +2렙 / 적당 → 유지 / 힘듦 → -2렙
- 특정 렙수 도달 + 연속 "쉬움" → 다음 레벨 동작으로 자동 진급

### 3. 히어로 랭크 시스템
- **C급** → **B급** → **A급** → **S급** 히어로
- 총 운동 볼륨 + 동작 레벨로 랭크 결정
- 랭크업 시 애니메이션 연출

### 4. 오늘의 트레이닝 화면
- 매일 들어오면 **오늘 할 운동이 바로 보임**
- 세트 카운터 + 휴식 타이머
- 운동 중 조작 최소화 (탭/스와이프)

### 5. 기록/통계 대시보드
- 주간/월간 볼륨 추이 그래프
- 트랙별 레벨 진행률
- 연속 운동 스트릭

## 🏗️ 기술 규칙

### 폴더 구조
```
src/
├── components/
│   ├── training/      # 오늘의 트레이닝, 세트 카운터, 타이머
│   ├── progression/   # 프로그레션 트리, 레벨업 UI
│   ├── rank/          # 히어로 랭크, 랭크업 연출
│   ├── stats/         # 기록, 통계, 대시보드
│   └── ui/            # 공통 UI (버튼, 카드, 모달 등)
├── stores/            # zustand 스토어
├── lib/               # firebase 클라이언트, 유틸리티
├── types/             # TypeScript 타입 정의
├── hooks/             # 커스텀 훅
└── docs/
    └── CHANGELOG.md
```

### 상태관리
- **zustand** 사용 (Redux 금지)
- 스토어는 `src/stores/` 하위에 기능별 분리
- localStorage persist (zustand/middleware)로 기본 저장
- 서버 상태(Firebase)와 클라이언트 상태 분리

### Firebase
- 클라이언트: `src/lib/firebase.ts`
- 환경변수: `VITE_FIREBASE_API_KEY`, `VITE_FIREBASE_AUTH_DOMAIN`, `VITE_FIREBASE_PROJECT_ID` 등
- Security Rules UID 기반 필수 적용
- Auth (Google 로그인) + Firestore (운동 기록 동기화)

### 스타일링
- **Tailwind CSS v4** (@tailwindcss/vite 플러그인)
- CSS 변수로 디자인 토큰 관리 (`src/index.css`)
- 인라인 스타일 금지, className으로 통일

### 코드 규칙
- 주석/커밋: 한국어
- 컴포넌트: 함수형 + TypeScript (FC 타입 지양, Props 인터페이스 명시)
- 파일명: kebab-case (`training-card.tsx`)
- 타입: `src/types/`에서 중앙 관리, `any` 금지

## 🚨 절대 규칙

1. **`any` 타입 금지** — 반드시 정확한 타입 사용
2. **Firebase Security Rules 없이 배포 금지**

4. **3개 이상 파일 동시 수정 시 확인**
